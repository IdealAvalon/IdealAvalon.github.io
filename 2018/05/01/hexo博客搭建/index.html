MacOS下的hexo个人博客搭建

Posted on 2018-05-01 | Edited on 2018-05-29 | In hexo | Views:

hexo博客搭建

本文从零开始搭建hexo博客，使用的系统是MacOS

内容简介

  • git 安装
  • node.js 安装
  • hexo 安装
  • github 相关使用
  • next主题 下载和简单设置
  • 个人域名设置（可选）
  • 创建博文

其中git,node.js以及GitHub是使用hexo搭建博客的基础，下面按照上述内容的顺序进行安装和操作

安装git

下载并安装git，点击git官方网站自行下载和安装
安装后打开终端，输入以下命令来验证git是否安装成功

$ git version
git version 2.13.1

会看到返回git的版本信息，说明安装成功

安装node.js

下载并安装node.js，点击node.js官方网站自行下载和安装
安装后打开终端，输入以下命令，来验证node.js安装是否成功

$ node -v
v8.11.1

可以看到返回node.js的版本信息，说明安装成功

安装hexo

上面我们安装了node.js，在这里使用node.js自带的npm工具安装hexo，只需要在终端中输入以下命令

$ npm i -g hexo

等待安装完毕即可,如果有权限问题，在命令前加sudo即可
接着输入以下命令，来测试hexo是否安装成功

$ hexo -v
hexo-cli: 1.1.0
os: Darwin 17.5.0 darwin x64
http_parser: 2.8.0
node: 8.11.1
v8: 6.2.414.50
uv: 1.19.1
zlib: 1.2.11
ares: 1.10.1-DEV
modules: 57
nghttp2: 1.25.0
openssl: 1.0.2o
icu: 60.1
unicode: 10.0
cldr: 32.0
tz: 2017c

可以看到输出很多信息，说明hexo安装成功

创建本地hexo文件夹

经过上面的过程，hexo的环境准备就结束了，现在开始正式搭建hexo博客

在自己喜欢的地方创建一个文件夹，以后我们博客的所有相关内容就都放到这个文件夹中了，笔者在~/Documents下创建了hexo文件夹
进入hexo文件夹下

$ cd ~/Documents/hexo

输入命令

$ hexo init

这样，hexo文件夹的初始化就成功了
进入hexo文件夹中，会发现多出来一些文件目录和文件，这里说明几个常用文件及目录的作用

  • _config.yml: 用来配置hexo的相关设置
  • source：用来存放创建的博文
  • public：用来存放生成的页面

到这里，我们的博客本地部分就已经搭好了，现在已经可以在本地访问，下面进行本地访问测试，输入命令：

$ sudo npm install hexo-server

来安装本地服务
然后生成静态页面，并启动本地服务：

$ hexo generate（或简写 hexo g）
$ hexo server

启动服务后，打开浏览器，按照终端的输出信息输入地址，一般是http://localhost:4000
如果没什么问题的话，就能访问到hexo的初始化页面,结束在终端中用：ctrl+c组合键

建议以后在将博文正式上传到GitHub之前先在本地预览，确定没问题，再上传。

关联github

关联GitHub后，我们便可以将博客相关内容上传到GitHub仓库，并在公网访问

注册GitHub账户并创建远程仓库

如果没有GitHub账户，首先创建GitHub账户
然后创建GitHub仓库，与hexo关联的仓库命名格式有要求：[yourname].github.io

配置hexo本地设置

创建好GitHub仓库后，配置本地信息，将本地hexo仓库与远程仓库进行连接
修改hexo根目录下的_config.yml文件,找到deploy属性设置 

deploy:
      type: git
      repository: git@github.com:yourname/yourname.github.io.git
      branch: master

注意.yml文件有固定的格式，在key的:后要加一个空格，否则会出错

配置本地git仓库

$ git config --global user.name "your name"
$ git config --global user.email "your email"

创建SSH

将本地ssh公钥复制到GitHub中就可以不输入用户名密码了

$ ssh-keygen

按三次回车，不用输入任何东西
找到公钥文件，并将文件复制到GitHub 

$ cd ~/.ssh
$ cat id_rsa.pub

然后将输出的内容全部复制
进入GitHub页面->settings->SSH and GPG keys-> new ssh
将内容全部复制进去即可

安装hexo的git部署

将本地hexo内容发布到GitHub仓库

$ npm install hexo-deployer-git --save

生成静态页面：

$ hexo generate

部署到GitHub远程仓库：

$ hexo deploy

如果这里报错：拒绝链接，首先使用sudo执行上面的命令，如果还报错，考虑与GitHub的ssh链接有问题，可以试试以下命令： 

$ ssh-add ~/.ssh/id_rsa

也就是把你的私钥添加到ssh认证中
或者上面两步用一个命令：

$ hexo d -g

公网访问博客

如果上述步骤没有问题的话，就可以通过如下网址访问博客了：
http://yourname.github.io

next主题

next主题是hexo主题中十分受欢迎的一个主题，网上也有很多关于next主题优化的博文可以参考
这里只做简单介绍

进入hexo/themes文件目录下，输入命令，下载next主题：

$ git clone https://github.com/theme-next/hexo-theme-next themes/next

然后编辑hexo根目录下的_config.yml文件
找到theme配置，并改为next
执行命令：

$ hexo clean
$ hexo g
$ hexo d

即可更换主题
详细的美化方案请自行搜索

绑定个人域名

如果有公网域名可以将博客的地址绑定到自己的公网域名，可以到阿里云购买，便宜的十几块钱就可以搞定

在这里笔者以阿里云购买的域名介绍

域名解析设置

登录阿里云网站,依次点击 控制台->域名与网站（万网）->云解析DNS->选择自己的域名->添加解析->

记录类型：A
主机记录：www.blog.steinsgate.cc（这里我使用了二级域名）
记录值：192.30.252.153（固定的，GitHub官方给了两个，这是其中一个）
点击确认

hexo配置设置

然后进入到hexo/source文件夹下，创建名称为CNAME的文件，没有任何扩展名

$ vim CNAME

在文件中写入www.blog.steinsgate.cc，即你在阿里云设置好的域名
注意：
由于mac系统的原因，这个文件在上传到GitHub仓库时，会变成cname，即小写，这样会导致域名绑定失败，所以要进行下面的设置
进入hexo/.deploy_git/文件目录下，修改.git文件
设置ignorecase = false
这样，在上传CNAME文件时，就不会变为小写（这是个坑，搞了好久才弄好）

测试域名是否绑定成功

在浏览器中输入你绑定的域名即可！

创建博文

在hexo根目录下输入命令： 

$ hexo new “文章名字”

进入hexo/source/_posts目录下，就可以看到新生成的文件了文章名字.md

编写博文

hexo博文编写使用markdown语法，可以下载macdown软件，对.md扩展名的文章进行编辑，详细请参阅markdown语法。
编写完成后执行命令 

$ hexo d -g

几秒钟后刷新网页就可以看到更改了
