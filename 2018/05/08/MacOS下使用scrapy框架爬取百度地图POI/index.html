scrapy框架优势

关于scrapy框架开发爬虫的官方话我就不说了，说一下自己感觉使用scrapy框架开发爬虫比使用诸如urllib库开发相比的几个优点

  • scrapy框架有自己的scheduler，即调度器，不需要手动写调度代码即可自动爬取多个地址
  • scrapy框架有pipeline组件，即管道下载器，下载数据的代码也不用写啦
  • 爬取地址参数中带有中文，不需要手动解码
  • 还有一个很好的功能，缓存，像是爬取百度地图数据，需要用到百度地图API，而对于开发者，百度地图API（应用程序接口）是有调用次数限制的，打开scrapy缓存功能后，爬取相同的地址则不需要去请求百度地图的API
  • 总之好处多多，笔者只是用了部分功能

Anaconda 介绍和使用

Anaconda是类似于pip的python依赖管理软件，管理（安装、卸载等）python依赖很方便

Anaconda几乎是一键傻瓜式安装，请点击Anaconda官网，自行下载和安装
安装有图形化和非图形化两种，看喜好，图形化安装包大很多，但没多大用就是了
安装完毕后，在终端输入conda（注意不是anaconda！）

$ conda --version
conda 4.4.10

看到返回信息，说明安装成功

scrapy 框架安装

有了Anaconda，只需要一条命令，就可以安装scrapy框架了，而且不需要配置环境变量 

$ conda scrapy

然后根据提示，输入y，确认安装即可
安装完毕后，在终端输入： 

$ scrapy

能返回一堆信息，包括scrapy框架的版本信息Scrapy 1.5.0 - no active project，说明安装成功

使用Pycharm开发工具导入和调试scrapy爬虫项目

这不是创建scrapy项目的必须步骤，不使用该IDE的可以跳过此步

由于不能直接在Pycharm IDE中创建爬虫项目，我们需要在终端中先创建scrapy爬虫项目，然后再导入到Pycharm中进行编辑

###在终端创建scrapy爬虫项目
首先创建目录专门用来保存爬虫项目，我这里已经有目录/Users/apple/PycharmProjects用来存放所有python项目，所以就直接进入到该目录下，并创建项目：

$ cd /Users/apple/PycharmProjects
$ scrapy startproject firstSpider
New Scrapy project 'firstSpider', using template directory '/Users/apple/anaconda3/lib/python3.6/site-packages/scrapy/templates/project', created in:
/Users/apple/PycharmProjects/firstSpider
You can start your first spider with:
cd firstSpider
scrapy genspider example example.com

返回上面一堆信息说明项目创建成功，框架提示我们可以再输入命令创建模板，这里我们不使用模板
这样一个scrapy框架爬虫项目就初始化成功了，下面为了编程方便，我们将其导入到IDE中

将创建好的scrapy项目导入到Pycharm中

打开Pycharm->open->找到项目目录->点击open，这样就导入成功了
下面我们介绍scrapy项目的结构

创建调试入口文件

在当前的IDE下，我们是无法运行和调试scrapy项目的，需要在项目的根目录添加一个入口文件
在根目录下创建文件entrypoint.py
写入代码： 

# -*- coding:UTF-8 -*-
from scrapy.cmdline import execute
# 下面两条语句二选一，第一条执行后会生成json格式的文件，第二条会生成excel表格文件
#execute(['scrapy','crawl','tongjiAnalyze'])
execute("scrapy crawl tongjiAnalyze -o tongji2.csv -t csv".split())

这样，在项目的编写过程中，就可以通过运行或调试此文件运行项目或进行调试

scrapy项目结构

scrapy框架目录结构如下：

├── firstSpider
│   ├── __init__.py
│   ├── __pycache__
│   ├── items.py
│   ├── middlewares.py
│   ├── pipelines.py
│   ├── settings.py
│   └── spiders
│       ├── __init__.py
│       └── __pycache__
└── scrapy.cfg
  • items.py：用来保存我们要爬取的数据的字段
  • pipelines.py：用来进行数据下载
  • settings.py：用来配置框架
  • spiders目录：在此目录下创建我们的爬虫文件
  • 其它文件不需要改动

scrapy项目实战，爬取百度地图POI

需求如下：

  • 以同济大学为中心，爬取周边美食、酒店的信息

获取百度地图API的秘钥

要爬取百度地图上的POI信息，必须要有百度地图API的ak秘钥串

登录百度地图api的官方网站
依次选择控制台(在页面右上角)->创建应用
应用名称：自己随意
应用类型：选择浏览器端
IP白名单：填写0.0.0.0/0
点击提交按钮后，应用创建成功，再点击页面左侧的查看应用，就可以看到应用的ak秘钥串

学习百度地图API的使用

在百度地图api官方网站右上角找到开发文档->服务接口->Web服务API
进入页面后选择左侧菜单栏中的地点检索->地点检索v2.0
再选择横向菜单栏中的服务文档，这里就提供了API的示例，我们向下翻，找到周边检索的接口： 

http://api.map.baidu.com/place/v2/search?query=银行&location=39.915,116.404&radius=2000&output=xml&ak=您的密钥 //GET请求

接口的参数信息在文档中有详细的介绍，不过我们不需要很多参数
在后面的爬虫程序请求中我们就使用这个接口
建议读者将上面的接口地址复制到浏览器地址栏，查看返回的json格式数据，以验证接口的可用性

在items.py中定义字段

我们想要从百度地图中获取POI点的名称、纬度、经度、地址信息，所以按如下定义：

在spiders文件目录下创建爬虫文件

在spiders目录下创建名为tongjiAnalyze.py的文件，开始我们的爬虫编写

# -*- coding:UTF-8 -*-

import scrapy
import json
from scrapy.http import Request
from coordTransform_utils import bd09_to_wgs84
import time

from tonjiAnalyze.items import TonjianalyzeItem


class TongjiAnalyze(scrapy.Spider):
    # 爬虫的名字，很重要！我们在运行爬虫任务的entrypoint文件中执行命令的第三个参数就是这个
    name = 'tongjiAnalyze'
    # 允许爬取的范围，这里不设置即可
    # allowed_domains=['http://api.map.baidu.com/']
    # 通过前面的百度地图API，我们要改变的参数为断点，将接口切分为三段，方便后面的拼接
    url0='http://api.map.baidu.com/place/v2/search?query='
    url1='&location=31.289,121.508&radius=1000&output=json&page_size=20&page_num='
    url2='&ak=xxxxxxxxxxxxxxxxxxxxxxx'
    # 需要爬取得分类，可以任意增删
    categories=['酒店','美食']
    # 该函数作用为初始化爬取地址，固定名称，不可修改！
    # 如果地址固定，也可以使用列表来代替此函数,同样，列表的名称不可修改：start_urls
    def start_requests(self):
        # 使用for循环来拼接地址，需要注意，页码是从0开始的
        for category in self.categories:
            url=self.url0+category+self.url1+str(0)+self.url2
            print('待爬取的产业分类为：'+category+',URL地址为：' + url)
            # 返回回调函数，来解析url地址
            yield Request(url,self.parse,meta={'category':category})
    # 回调函数，用于解析start_requests中的url地址
    def parse(self, response):
        # 获取第一页json数据，用来获取这个分类下的total值
        hjson = json.loads(response.text)
        # 根据total值来获取需要爬取的总页数
        max_page=int(hjson['total']/20)
        print('分类：'+response.meta['category'] + ',总共需要爬取的页数为：' + str(max_page))
        for i in range(0, max_page):
            # 拼接新的要爬取的url
            url = self.url0 + response.meta['category'] + self.url1 + str(i) + self.url2
            print('正在爬取第{}页，url地址为{}'.format(i+1,url))
            yield Request(url,self.parse2)
            # 防止并发太高，百度地图API对并发量有限制
            time.sleep(0.5)
    # 自定义回调函数，用来将跟进的url的数据发送给管道，以进行存储
    # 需要注意的是，自定义回调函数，即使在测试的时候也要有 “yield” 代码，否则Request函数不认为它是一个可回调的函数
    def parse2(self,response):
        hjson = json.loads(response.text)
        if hjson['message'] == 'ok':
            results = hjson['results']
            for result in results:
                item = TonjianalyzeItem()
                item['name'] = result['name']
                item['lat'] = result['location']['lat']
                item['lng'] = result['location']['lng']
                item['address'] = result['address']
                item['wgs84_lng_lat']=bd09_to_wgs84(result['location']['lng'],result['location']['lat'])
                yield item

编写pipelines.py文件

# -*- coding: utf-8 -*-
import json
# Define your item pipelines here
#
# Don't forget to add your pipeline to the ITEM_PIPELINES setting
# See: https://doc.scrapy.org/en/latest/topics/item-pipeline.html


class TongjianalyzePipeline(object):
    def __init__(self):
        # 打开文件
        self.filename = open('/Users/apple/Desktop/testJson.txt', 'w', encoding='utf-8')

    # 该方法用于处理数据
    def process_item(self, item, spider):
        # 读取item中的数据
        line = json.dumps(dict(item), ensure_ascii=False) + "\n"
        # 写入文件
        self.filename.write(line)
        # 返回item
        return item

程序运行结束后会发现根目录下产生相应的数据文件。
