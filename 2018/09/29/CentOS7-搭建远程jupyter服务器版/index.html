CentOS7 搭建远程jupyter服务器版

Posted on 2018-09-29 | In python | Views:

在阿里云上安装jupyter，并开启远程访问

安装Anaconda

下载Anaconda

使用wget命令在官网地址下载即可

安装Anaconda

$ bash ./Anaconda3-5.2.0-Linux-x86_64.sh

注意：在安装过程中会有两次询问，第一次询问是否添加到环境变量，输入yes
第二次就有推销的嫌疑了，询问是否安装VSCode，果断输入no

测试Anaconda是否安装成功

$ conda -V
conda 4.5.11

显示conda的地版本信息表示成功

如果显示command not found,可能是添加环境变量没有成功，则需要手动添加

手动添加环境变量

编辑profile文件

$ vim /etc/profile
# 用你自己的路径替换
ANACONDA_HOME=/root/apps/anaconda3
export PATH=$PATH:$ANACONDA_HOME/bin

然后在终端输入

conda -V

这样应该就能成功了

测试本地python3是否是Anaconda中的

如果服务器原来就安装过python3，那么安装完anaconda后，默认还是使用本地的python3环境，这样有可能导致anaconda中安装的包不能使用

测试python3环境

$ python3
Python 3.6.5 |Anaconda, Inc.| (default, Apr 29 2018, 16:14:56) 
[GCC 7.2.0] on linux

像上面，如果python版本后面带有Anaconda，则说明默认使用anaconda中的python3环境，可以跳过本小节的内容，否则，需要修改默认环境

修改python3默认环境

进入/bin目录下，查看python3软链接

$ cd /bin
$ ll python*

则可以看到python3的软链接位置，如果不是/root/apps/anaconda3/bin/python3,则重新建立软链接覆盖原有链接

$ rm -rf python3
$ ln -s /root/apps/anaconda3/bin/python3 /bin/python3
  • 注意是删除了/bin目录下的python3
  • 建立软链接一定要使用绝对路径！而且源文件和目标文件的顺序一定不要弄反！

这样再次在终端输入python3应该使用的就是anaconda自带的python3环境了

使用Anaconda安装jupyter

使用命令：

$ conda install jupyter

即可安装

测试jupyter是否安装成功：

$ jupyter notebook -V
5.5.0

显示版本号，即安装成功

设置jupyter密码

在终端输入下面的命令，来生成远程访问的密码：

# python3
>> import notebook.auth import passwd
>> passwd()
Enter password：
Verify password:
'sha1:·········'

将生成的密钥’sha1……’复制下来，用于后面的配置文件修改

为jupyter开放远程访问

1.首先生成jupyter配置文件

$ jupyter notebook --generate-config

执行完上面的命令后，会生成配置文件：

~/.jupyter/jupyter_notebook_config.py

2.编辑jupyter配置文件

$ vim  ~/.jupyter/jupyter_notebook_config.py
# 这些配置信息原本都存在，只不过被注释掉了，为了方便，我们直接在文件最后添加这些配置即可
c.NotebookApp.allow_root = True   # 开放root用户使用
c.NotebookApp.ip='*'              # 所有ip均可访问
c.NotebookApp.password = u'sha:ce...刚才复制的那个密文'
c.NotebookApp.open_browser = False      # 禁止启动jupyter时打开浏览器
c.NotebookApp.port =8888          #可自行指定一个端口, 访问时使用该端口

保存并退出

设置Linux防火墙

开放8888端口 

$ firewall-cmd --add-port=8888/tcp --permanent
$ firewall-cmd --reload
  • 永久开放8888端口
  • 重新载入防火墙规则，使上面的命令生效

我的服务器是使用的阿里云的服务器，不要忘记把阿里云的防火墙也开放8888端口，这里我就不介绍了

守护进程启动jupyter

使用nohup命令创建jupyter的守护进程

$ nohup jupyter notebook >> /root/apps/logs/jupyter.log 2>&1 &
  • nohup <进程名> & 为启动守护进程
  • >> /root/apps/logs/jupyter.log 表示将启动日志追加到jupyter.log文件中
  • 2>&1 表示将标准出错重定向到标准输出，即将标准出错也输出到jupyter.log文件中

最后，从浏览器中访问IP:8888然后输入密码就可以使用了
