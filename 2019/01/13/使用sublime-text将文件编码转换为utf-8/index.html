使用sublime text将文件编码转换为utf-8

Posted on 2019-01-13 | In 工具 | Views:

使用背景

由于将表格类数据导入python中进行数据分析的时候非utf8的编码有时候会不能很好地识别，而使用excel打开大文件（50M以上可能就会卡了）进行转码又很费劲，所以想着使用轻便的sublime进行转码

使用环境

  • macos
  • sublime text

软件和插件的安装

需要安装的软件和插件：

  • sublime text
  • 插件：Package Control
  • 插件：ConvertToUTF8
  • 插件：codecs33

安装结束后重启sublime text，点击 sublime text - Preferences
应该就有了Package Settings 和 Package Control 两个选项

安装ConvertToUTF8插件

  1. 可以点击上面步骤中的Package Control选项，也可以使用快捷键：command + shift + p
    打开Package Control搜索框
  2. 输入install package 并回车，调出instal package 输入框
  3. 在新的输入框中输入 ConverToUTF8，并回车，安装插件

安装codecs33插件

在网上参考的资料中看到，有些用户进行到上面一步，就可以成功打开非utf8的文件了，但是我这里不行，根据ConvertToUTF8插件的提示，要再安装codecs33插件

  2. 在输入框中，输入 codecs33， 并回车，安装插件

测试打开win系统中拷贝过来的csv文件

如果文件较大，可能打开后需要几秒钟的转换时间，50M的文件应该不超过10秒钟
如果能正确显示，就ok了

转换成utf8格式

提示，最好是把源文件复制一份，因为转换时不会生成新文件

依次点击 file - set file encoding to , 然后选择utf8就好了

这里亲测，如果使用sublime text自带的save with encoding选项，是不能成功保存为utf8格式的

PS: 卸载插件

command + shift + p
输入remove package 回车，就能看到已经安装的插件了，然后选择插件，回车，即可卸载
